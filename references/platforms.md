# Platform Formatting Guide

## X / Twitter Thread

- **Length**: 8-12 tweets optimal (47% better engagement than shorter)
- **Per tweet**: ≤280 characters
- **Numbering**: Use "1/10" format for completion motivation
- **Visuals**: Insert image/screenshot every 3-4 tweets
- **First tweet**: Hook only. No context, no intro, just the hook
- **Last tweet**: CTA — "Follow for more" / "RT if useful" / question
- **Timing**: Tue/Wed 8-10 AM or 7-9 PM EST; 中国受众: 晚8-10点 CST
- **Post-publish**: Reply to every comment in first 30 min (algorithm reward)
- **Format**: No markdown tables. Use line breaks and emoji bullets

## 公众号 (WeChat Official Account)

- **Length**: ~2000 characters for stories, ~3000 for deep dives
- **Structure**: Scene-setting opener → conflict → insight → golden quote ending
- **Golden lines**: Write 2-3 "screenshot-worthy" quotes readers will share
- **Headers**: Use emoji + bold for section headers
- **Rhythm**: Alternate long analytical paragraphs with short punchy lines
- **Images**: Cover image is crucial; use mid-article images to break reading fatigue
- **CTA**: End with "你觉得呢？" or similar engagement prompt

## LinkedIn

- **Tone**: Professional but personal (not corporate)
- **Structure**: Short hook line → line break → story → takeaway
- **Length**: 1300-1500 characters optimal
- **Paragraphs**: Max 2-3 lines each, heavy line breaks
- **Hashtags**: 1-3 at the very end, relevant ones only
- **No**: Emojis in every line (unlike X), no "THREAD 🧵"
- **Yes**: "Agree?" / "What's your experience?" as CTA

## 小红书 (Xiaohongshu / RED)

- **Visual-first**: Cover image determines 70% of clicks
- **Tone**: Relatable, casual, 闺蜜/朋友 vibes
- **Length**: 300-800 characters (shorter than other platforms)
- **Emoji**: Heavy use, as section markers and emphasis
- **Structure**: Problem → my experience → solution/tips → tags
- **Tags**: 5-10 relevant tags at the end
- **CTA**: "收藏💛" / "你们遇到过吗？"

## Discord / WhatsApp

- **No**: Markdown tables (they don't render properly)
- **Yes**: Bullet lists, bold text, emoji
- **Links**: Wrap in `<>` on Discord to suppress embeds
- **Length**: Keep messages under 2000 chars; split at logical breaks
- **Headers**: Use **bold** or CAPS for emphasis (no # headers on WhatsApp)

## Newsletter / Email

- **Subject line**: Treat like a hook — same formulas apply
- **Preview text**: First 40-60 chars after subject, make them count
- **Structure**: Personal opener → main content → one clear CTA
- **Length**: 500-1500 words for weekly; 200-500 for daily
- **P.S.**: Always add a P.S. line — highest-read section after subject line

## Cross-Platform Distribution Strategy

```
Original → X Thread (test engagement)
   ↓
High engagement? → Expand to 公众号 (deeper version)
   ↓
Professional angle? → Adapt for LinkedIn
   ↓
Visual angle? → Adapt for 小红书
   ↓
Best performers → Compile into Newsletter
```
